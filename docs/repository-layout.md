# Repository layout

This document describes the main repository paths that contributors need when
working on `falcon-correlate`. It is the canonical location for file-tree and
path-responsibility guidance.

## Tree overview

This tree is a compact orientation sketch and omits generated caches,
virtual-environment directories, and build artefacts.

```plaintext
.
├── .github/
├── .rules/
├── docs/
│   └── execplans/
├── src/
│   └── falcon_correlate/
│       └── unittests/
├── tests/
│   ├── bdd/
│   ├── property/
│   └── workflows/
├── AGENTS.md
├── Makefile
├── pyproject.toml
└── uv.lock
```

_Figure 1: Repository tree overview for the maintained source and documentation
paths._

## Path responsibilities

| Path                              | Responsibility                                                                                          |
| --------------------------------- | ------------------------------------------------------------------------------------------------------- |
| `AGENTS.md`                       | Defines repository-specific agent, documentation, quality-gate, and commit rules.                       |
| `Makefile`                        | Provides the canonical local build, formatting, linting, typechecking, documentation, and test targets. |
| `pyproject.toml`                  | Defines project metadata, dependencies, package configuration, and Python tool settings.                |
| `uv.lock`                         | Pins the resolved Python dependency graph used by `uv`.                                                 |
| `.github/`                        | Stores GitHub Actions workflows and local composite actions used by repository automation.              |
| `.rules/`                         | Holds Python-specific coding guidance referenced by `AGENTS.md` and maintainer documentation.           |
| `docs/`                           | Contains long-lived user, developer, design, decision, reference, and planning documentation.           |
| `docs/execplans/`                 | Stores task-level execution plans for implementation work.                                              |
| `src/falcon_correlate/`           | Contains the installable Python package and public middleware implementation.                           |
| `src/falcon_correlate/unittests/` | Contains package-local unit tests that exercise implementation units close to the source.               |
| `tests/bdd/`                      | Contains behavioural tests written from user-observable workflows.                                      |
| `tests/property/`                 | Contains Hypothesis-based property tests for invariants that benefit from generated inputs.             |
| `tests/workflows/`                | Contains workflow-level tests and fixtures that validate repository automation behaviour.               |

_Table 1: Maintained repository paths and their responsibilities._

## Placement conventions

- Put user-facing behaviour, examples, configuration rules, and public API
  guidance in [the user's guide](users-guide.md).
- Put contributor workflows, quality gates, implementation conventions, and
  maintainer-facing operational notes in
  [the developers' guide](developers-guide.md).
- Put architecture, constraints, rationale, and design evolution in
  [the design document](falcon-correlation-id-middleware-design.md).
- Put accepted decisions in Architecture Decision Records (ADRs) under
  `docs/`, using the naming convention from
  [the documentation style guide](documentation-style-guide.md).
- Put pre-implementation or task-level plans in `docs/execplans/`.
- Put Python package code under `src/falcon_correlate/`; avoid placing runtime
  package modules under `tests/` or repository-root helper paths.
- Put user-observable workflow coverage under `tests/bdd/`, invariant coverage
  under `tests/property/`, and automation coverage under `tests/workflows/`.

## Generated and local-only paths

The following paths are local artefacts and should not be treated as source of
truth:

- `.venv/` contains the local virtual environment created by `uv`.
- `.uv-cache/` and `.uv-tools/` contain project-local `uv` caches and tools.
- `.ruff_cache/`, `.pytest_cache/`, `.mypy_cache/`, `htmlcov/`, `build/`, and
  `dist/` are generated by quality gates, coverage, or packaging commands.

Do not document durable architecture or maintainer process only inside
generated or local-only paths. Promote durable information into the relevant
document under `docs/`.
